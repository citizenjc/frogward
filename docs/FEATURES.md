# Features

## Current features

- SAPO webmail login automation
- session reuse with saved browser state
- inbox parsing from the live SAPO UI
- new-mail detection with local state
- sender/subject allow + block filters
- one-shot forwarding mode
- always-on service mode
- retry-visible failure tracking
- success confirmation with Sent-folder fallback
- local failure artifacts for debugging

## Current boundaries

- no spam-folder scanning yet
- no hosted/cloud control plane
- no database
- no multi-user setup
- no guaranteed toast-based confirmation from SAPO UI

## Intended audience

- personal/home-server users
- self-hosters
- users who need SAPO mail forwarded to a more reliable inbox
